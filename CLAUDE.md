# CLAUDE.md

## Project Philosophy

This is a minimal Android development environment that avoids Android Studio entirely. Everything runs from the command line and a text editor. The setup prioritizes:

- **No IDE.** No Android Studio. No GUI tools. Just a terminal and an editor.
- **No system pollution.** All dependencies live in the Nix shell or the local `.gradle/` directory. Nothing is installed system-wide except Nix itself.
- **Simplicity.** The project uses plain Java (not Kotlin), XML layouts, and Gradle for builds. No Jetpack Compose, no complex abstractions.
- **C-programmer mindset.** Edit, compile, install, debug with printf (`Log.d`). The Makefile is the entry point for all build operations.

## What this app is

♄ (Saturn) is the Android port of [nudgent](../nudgent): an intelligent task nudge assistant. The server/Telegram layer is replaced by a local chat UI and Android notifications, while inference still goes through the OpenRouter API.

Architecture:
- `MainActivity` — chat screen (user types, agent replies)
- `SettingsActivity` — API key, model, timezone, language, nudge interval, schedule
- `Database` — SQLite via SQLiteOpenHelper (tasks table)
- `AgentClient` — HTTP to OpenRouter + system prompt builders
- `ActionExecutor` — applies agent actions to the DB and SharedPreferences
- `NudgeService` — foreground Service that runs the nudge/schedule cycle
- `NudgeReceiver` — BroadcastReceiver triggered by AlarmManager
- `BootReceiver` — reschedules alarms after reboot
- `NudgeScheduler` — wraps AlarmManager setup

Settings are stored in SharedPreferences (`saturn`). The only external dependency is Gson for JSON parsing.

## Environment

You are running inside a **Nix shell** (`nix-shell`). The shell provides:

- **JDK 17** — Java compiler and runtime
- **Gradle** — build system
- **Android SDK** — platform-tools, build-tools 34.0.0, platform API 34

Key environment variables (set by `shell.nix`):

- `$ANDROID_HOME` / `$ANDROID_SDK_ROOT` — points to the Nix-managed Android SDK
- `$GRADLE_USER_HOME` — set to `.gradle/` (local to the project, not `~/.gradle`)

The file `local.properties` is auto-generated by the shell hook. Do not edit it manually or commit it.

## Build Commands

| Command        | What it does                              |
|----------------|-------------------------------------------|
| `make`         | Build debug APK                           |
| `make install` | Build and install on connected device     |
| `make run`     | Build, install, and launch the app        |
| `make release` | Build release APK                         |
| `make clean`   | Remove all build artifacts                |
| `make logcat`  | Stream filtered logs from the device      |

## Coding Guidelines

- **Java only.** No Kotlin, no Jetpack Compose.
- **No AndroidX.** All framework calls use `android.app.*`, `android.widget.*` etc. directly.
- **XML layouts.** Traditional XML in `res/layout/`. No programmatic UI unless justified.
- **Minimal deps.** Only Gson. Don't add libraries unless asked.
- **Logging.** `Log.d("Saturn", "message")` throughout.

## Things to Avoid

- Do not suggest or use Android Studio, IntelliJ, or any IDE.
- Do not run `gradle` directly — use Makefile targets.
- Do not modify `local.properties`.
- Do not add Gradle wrapper files.
- Do not install anything globally.
- Do not add AndroidX dependencies.
